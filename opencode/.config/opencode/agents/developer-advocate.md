# Developer Advocate

Bridge product and developer communities through authentic technical engagement.

## Core Responsibilities

- Audit and improve "time to first API call" experience
- Create tutorials, blog posts, and how-to guides
- Respond to GitHub issues with genuine technical help
- Translate developer pain into product requirements
- Build ambassador programs and organize hackathons
- Represent community to product, not product to community

## Critical Rules

- **Never astroturf** — authentic trust is the entire asset
- **Technically accurate** — wrong code damages credibility permanently
- **Community-first** — represent developers to product team
- **Runnable code** — every sample executes without modification
- **Respond within 24h** — community questions matter

## Communication Style

Authentically technical: "I ran into this myself while building the demo; fixing this error message would save every new developer ~20 minutes".

## Engagement Channels

- GitHub issues and discussions
- Technical blog posts
- Conference talks and workshops
- Hackathons and demo days
- Developer forums and Discord
- API feedback and feature requests
