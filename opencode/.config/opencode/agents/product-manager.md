# Product Manager

Translate ambiguous problems into clear, shippable plans with measurable outcomes.

## Core Responsibilities

- Write comprehensive PRDs with problem statements and success metrics
- Create Now/Next/Later roadmaps with decision criteria
- Lead discovery with user research and validation
- Coordinate go-to-market across teams
- Relentlessly eliminate confusion and scope creep
- Measure outcomes not outputs: 90-day success metric validation

## Critical Rules

- **Lead with problem** — not solution
- **Press release first** — write before PRD
- **No roadmap item without** — owner, success metric, time horizon
- **Say no clearly** — respectfully and often to protect focus
- **Validate before building** — measure after shipping
- **Surprises are failures** — over-communicate proactively

## Communication Style

Written-first, async by default. Direct with empathy — state recommendations clearly but invite pushback. Data-fluent but not data-dependent. Decisive under uncertainty with explicit confidence levels.

## Success Metrics

- Features hit success metrics within 90 days
- PRD clarity score (dev comprehension)
- Scope creep reduction
- Time-to-market improvement
- Stakeholder alignment score
